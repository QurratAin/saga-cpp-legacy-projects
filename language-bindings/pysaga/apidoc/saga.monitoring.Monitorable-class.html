list_metrics(self)

 
List all metrics associated with the object.
Returns: tuple
names identifying the metrics associated with the object instance
Raises:
  • NotImplemented -
  • PermissionDenied -
  • AuthorizationFailed -
  • AuthenticationFailed -
  • Timeout -
  • NoSuccess -

Permission: Query

Notes:
  • several SAGA objects are required to expose certain metrics (e.g. "task.state"). However, in general that assumption cannot be made, as implementations might be unable to provide metrics. In particular, listed metrics might actually be unavailable.
  • no order is implied on the returned tuple
  • the returned tuple is guaranteed to have no double entries (names are unique)
  • an "AuthenticationFailed", "AuthorizationFailed" or "PermissionDenied" exception indicates that the current session is not allowed to list the available metrics.
  • a "Timeout" or "NoSuccess" exception indicates that the backend was not able to list the available metrics.

Returns: Metric
metric instance identified by name
Raises:
  • NotImplemented -
  • DoesNotExist -
  • PermissionDenied -
  • AuthorizationFailed -
  • AuthenticationFailed -
  • Timeout -
  • NoSuccess -

Permission: Query

Notes:
  • multiple calls of this method with the same value for name return multiple identical instances (copies) of the metric.
  • a "DoesNotExist" exception indicates that the backend does not know the metric with the given name.
  • an "AuthenticationFailed", "AuthorizationFailed" or "PermissionDenied" exception indicates that the current session is not allowed to obtain the named metric.
  • a "Timeout" or "NoSuccess" exception indicates that the backend was not able to return the named metric.
