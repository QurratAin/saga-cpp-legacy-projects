Class StdIO



object --+
         |
        StdIO
This class is used to give acces to the opaque data like from from stdin, stdout and stderr. This is an extention from the SAGA specification. StdIO supports most operations associated with stdin, stdout and stderr but not all methods are available in the same object. For instance: if a StdIO object represents the job's stdin, it makes no sense to read from the StdIO object as a stdin cannot generate data.

read(self, size=-1, blocking=True)

 
Read at most size bytes from stdout or from stderr, depending which one is represented by this object.
Parameters:
  • size (bool) - Maximum number of characters to read.
  • blocking (int) - whether the read should block until it has read a number of bytes equal to size or not.
Returns: string
string with the data
Raises:
  • PermissionDenied - if this object represents the stdin of the job.
Notes:
  • if blocking is False, read may return a string which length is smaller than parameter size.
  • if size is not specified read will return as much data as is available
  • if read has read an EOF from the stdout or stderr, read returns an empty string for the blocking case and None for the non-blocking case.

readline(self, size=-1, blocking=True)

 
Read the next line from the stdout or stderr of the job.
Parameters:
  • size (bool) - Maximum number of characters to read. An incomplete line may be returned
  • blocking (int) - whether the read should block until it has read a line or number of bytes equal to size or not.
Returns: string
string with the line of data including the newline character
Raises:
  • PermissionDenied - if this object represents the stdin of the job.
Notes:
  • if blocking is False, readline may return a string which length is smaller than parameter size.
  • if size is not specified readline will return one line of data or as much data as is available
  • if readline has read an EOF from the stdout or stderr, read returns an empty string for the blocking case and None for the non-blocking case.

readlines(self, size, blocking=True)

 
Read multiple lines from the stdout or stderr.
Parameters:
  • size (bool) - Maximum number of characters to read. An list with an incomplete line as last element of the list may be returned
  • blocking (int) - whether the read should block until it has read a number of bytes equal to size or not.
Returns: list
List of strings with lines of data without the newline characters
Raises:
  • PermissionDenied - if this object represents the stdin of the job.
Notes:
  • if blocking is False, readlines may return a list of strings which total length including newlines is smaller than parameter size.
  • if size is not specified readlines will return a list of as much strings available. The last element in the list may be an incomplete line
  • if readlines has read an EOF from the stdout or stderr, read returns an empty list for the blocking case and None for the non-blocking case.
