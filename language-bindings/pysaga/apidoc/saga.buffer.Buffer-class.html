The saga.buffer.Buffer class encapsulates a sequence of data to be used for I/O operations, that allows for uniform I/O syntax and semantics over the various SAGA API packages.

__init__(self, size=-1, data=None)
(Constructor)

 
Initialize an I/O buffer.
  • Call format: Buffer( size, data )
    • Precondition:
      • size must be >= 0
    • Postcondition:
      • the buffer memory is managed by the application.
  • Call format: Buffer( size )
    • Postcondition:
      • if "data" is not specified and size > 0, the buffer memory is allocated by the implementation.
      • if "data" is not specified, the buffer memory is managed by the implementation.
Parameters:
  • size (int) - size of data buffer to be used
  • data (char array or a list) - buffer to be used
Raises:
  • NotImplemented -
  • BadParameter - if the implementation cannot handle the given data parameter or the given size
  • NoSuccess -
Overrides: object.__init__

See Also: notes about memory management in GFD-R-P.90 document.

get_size(self)

 
Retrieve the current value for size.
Returns: int
value of size
Raises:
  • NotImplemented -
  • IncorrectState -
Notes:
  • if the buffer was created with negative size with the second call format of the __init__(), or the size was set to a negative value with set_size(), this method returns '-1' if the buffer was not yet used for an I/O operation.
  • if the buffer was used for a successfull I/O operation where data have been read into the buffer, the call returns the size of the memory which has been allocated by the implementation during that read operation

Notes:

  • the notes for __del__ and the first __init__ call format apply.

get_data(self)

 
Retrieve the buffer data.
Returns: char array, list, string
buffer data to retrieve. Type depends on what type was used to create the Buffer object. If the buffer is implementation managed, it returns a string.
Raises:

  • DoesNotExist -
  • IncorrectState -
Notes:
  • see notes about memory management in the GFD-R-P.90 document
  • if the buffer was created as implementation managed (size == -1), but no I/O operation has yet been successfully performed on the buffer, a 'DoesNotExist' exception is raised.

close(self, timeout=-0.0)

 
Closes the object.
Parameters:
  • timeout (float) - seconds to wait
Raises:
  • NotImplemented -

Postcondition: any operation on the object other than close() or __del__ will cause an 'IncorrectState' exception.

Notes:
  • if the current data memory is managed by the implementation, it is freed.
  • if the current data memory is managed by the application, it is not accessed anymore by the implementation after this method returns.
  • if close() is implicitly called in __del__(), it will never raise an exception.

See Also: for resource deallocation semantics and timeout semantics, see Section 2 of the GFD-R-P.90 document
